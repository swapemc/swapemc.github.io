Documentation

ImguiLib

A faithful Dear ImGui-styled UI library for Roblox Luau. Dense, flat, monospace — built for scripts, exploits, and developer tools. Now with a full config save/load system.

Luau Roblox Dear ImGui Style
01

Installation

Load the library at the top of your script using loadstring and HttpGet.

Luau
local ImGui = loadstring(game:HttpGet(
    "https://gist.githubusercontent.com/swapemc/5471fafc4ab020982e882d243fbbdf4d/raw/574f5fd2c654a9e0d20747290e3084bb56dea591/gistfile1.txt"
))()
ⓘ
The library attempts to parent its ScreenGui to CoreGui first (exploit environments), and falls back to PlayerGui automatically.
02

Quick Start

Create a window, add a tab, add widgets. Assign key strings to any widget you want to persist across sessions.

Luau
local ImGui = loadstring(game:HttpGet("..."))()

local Window = ImGui.CreateWindow("My Tool")
local Tab    = Window:AddTab("Main")

-- widgets with a key are saved/loaded automatically
Tab:AddCheckbox("God Mode", false, function(v) print(v) end, "godmode")
Tab:AddSliderInt("Walk Speed", 0, 200, 16, function(v) print(v) end, "walkspeed")

-- load saved config, then enable auto-save on every change
Window:LoadConfig("default")
Window:EnableAutoSave("default")

CreateWindow

Entry point of the library. Returns a Window object.

Signature
ImGui.CreateWindow(title: string, options?: table) → Window
ParameterTypeDescription
titlestringText shown in the title bar.
options.Widthnumber?Window width in pixels. Default: 430
options.Heightnumber?Window height in pixels. Default: 480
options.PositionUDim2?Initial screen position. Default: centered.
options.ToggleKeyKeyCode?Key to show/hide the window. Default: RightShift
Example
local Window = ImGui.CreateWindow("My Tool", {
    Width     = 500,
    Height    = 600,
    Position  = UDim2.new(0.5, -250, 0.5, -300),
    ToggleKey = Enum.KeyCode.Insert,
})
04

Window API

Methods available on the Window object. Config methods are covered in the Config API section.

method
Window:AddTab(name: string) → Tab

Creates a new tab in the menu bar and returns a Tab object. The first tab added is auto-selected. If more tabs are added than fit in the bar, the bar scrolls horizontally.

method
Window:SelectTab(name: string)

Programmatically switches to the tab with the given name.

method
Window:Toggle()

Toggles the window's visibility. Also triggered by the keybind.

method
Window:SetVisible(visible: boolean)

Sets visibility directly without toggling.

method
Window:SetToggleKey(key: Enum.KeyCode)

Changes the keybind used to show/hide the window at runtime.

method
Window:SetTheme(overrides: table)

Applies colour overrides. Only keys you provide are changed. See the Theming section.

method
Window:Destroy()

Removes the ScreenGui and cleans up the window entirely.

05

Tabs

Tabs are created on the Window and act as containers for all widgets. They render as buttons in the menu bar.

Example
local MainTab     = Window:AddTab("Main")
local CombatTab   = Window:AddTab("Combat")
local SettingsTab = Window:AddTab("Settings")

AddText

Renders a static text label. No key — text has no saveable state.

Signature
Tab:AddText(text: string, disabled?: boolean) → { SetText }
Example
Tab:AddText("Hello, world!")
Tab:AddText("Greyed out", true)
local lbl = Tab:AddText("Score: 0")
lbl:SetText("Score: 100")
07

AddButton

A clickable button. Width auto-fits to the label. No key — buttons have no saveable state.

Signature
Tab:AddButton(text: string, callback?: function) → { SetText, SetCallback }
Example
Tab:AddButton("Teleport", function() print("clicked") end)

AddCheckbox

A boolean toggle rendered as a small square checkbox.

Signature
Tab:AddCheckbox(text: string, default: boolean, callback?: function, key?: string)
    → { GetValue, SetValue, SetCallback }
Example
Tab:AddCheckbox("God Mode", false, function(v)
    local hum = game.Players.LocalPlayer.Character.Humanoid
    hum.MaxHealth = v and math.huge or 100
end, "godmode")

AddRadioGroup

A row of mutually exclusive radio buttons.

Signature
Tab:AddRadioGroup(items: table, default: string, callback?: function, key?: string)
    → { GetSelected, SetCallback }
Example
Tab:AddRadioGroup(
    {"radio a", "radio b", "radio c"},
    "radio a",
    function(v) print(v) end,
    "aimmode"

AddSliderInt / AddSliderFloat

A draggable slider. Value is printed inside the bar. AddSliderInt snaps to whole numbers; AddSliderFloat gives 3 decimal places.

Signatures
Tab:AddSliderInt(text: string, min: number, max: number, default: number, callback?: function, key?: string)
    → { GetValue, SetValue, SetCallback }

Tab:AddSliderFloat(text: string, min: number, max: number, default: number, callback?: function, key?: string)
    → { GetValue, SetValue, SetCallback }
Example
Tab:AddSliderInt("Walk Speed", 0, 200, 16, function(v)
    game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = v
end, "walkspeed")

Tab:AddSliderFloat("Gravity", 0.0, 5.0, 1.0, function(v)
    workspace.Gravity = v * 196.2
end, "gravity")

AddInputText

A single-line text input. Callback fires on focus lost. The second callback argument is enterPressed: boolean.

Signature
Tab:AddInputText(text: string, default: string, callback?: function, key?: string)
    → { GetText, SetText, SetCallback }
Example
Tab:AddInputText("Target", "", function(t, enter)
    if enter then print("Confirmed:", t) end
end, "target")

AddInputInt / AddInputFloat

Numeric inputs with − and + stepper buttons. The optional step parameter controls how much the steppers change the value.

Signatures
Tab:AddInputInt(text: string, default: number, callback?: function, step?: number, key?: string)
    → { GetValue, SetValue, SetCallback }

Tab:AddInputFloat(text: string, default: number, callback?: function, step?: number, key?: string)
    → { GetValue, SetValue, SetCallback }
Example
Tab:AddInputInt("Range", 20, function(v) print(v) end, 5, "range")
Tab:AddInputFloat("Scale", 1.0, function(v) print(v) end, 0.1, "scale")

AddCollapsingHeader

Returns a Section — call any widget method on it. Widgets inside pass their keys through to the same window-level registry, so they save/load normally.

Signature
Tab:AddCollapsingHeader(text: string, defaultOpen?: boolean) → Section
Example
local s = Tab:AddCollapsingHeader("Aim Settings", true)
s:AddCheckbox("Silent Aim", false, function(v) print(v) end, "silentaim")
s:AddSliderInt("FOV", 0, 360, 60, function(v) print(v) end, "fov")

18

Widget Keys

Every stateful widget accepts an optional key string as its last argument. The key is how the config system identifies each widget when saving and loading. Keys must be unique across the entire window.

⚠
Keys must be unique across all tabs and all collapsing headers in the same window. Using the same key twice will cause one widget to overwrite the other's saved value.
1
Add a key to any widgetPass a unique string as the last argument to AddCheckbox, AddSliderInt, AddSliderFloat, AddInputText, AddInputInt, AddInputFloat, AddCombo, AddColorEdit, or AddRadioGroup.
2
Call LoadConfig after adding all widgetsThis restores all saved values and fires each widget's callback with the loaded value.
3
Call EnableAutoSave or SaveConfig manuallyAuto-save fires SaveConfig on every widget change. Manual save gives you explicit control.
Widgets that support keys
Tab:AddCheckbox(text, default, callback,       key)
Tab:AddSliderInt(text, min, max, def, callback, key)
Tab:AddSliderFloat(text, min, max, def, callback, key)
Tab:AddInputText(text, default, callback,       key)
Tab:AddInputInt(text, default, callback, step, key)
Tab:AddInputFloat(text, default, callback, step, key)
Tab:AddCombo(text, options, default, callback,  key)
Tab:AddColorEdit(text, default, callback,       key)
Tab:AddRadioGroup(items, default, callback,     key)
19

Save & Load

Configs are stored as JSON files at ImGuiLib/<name>.json in your executor's filesystem. You can have multiple named configs — useful for presets.

Example
-- Save the current state of all keyed widgets
Window:SaveConfig("default")   -- → ImGuiLib/default.json
Window:SaveConfig("preset_a") -- → ImGuiLib/preset_a.json

-- Load and apply a saved config
-- Returns true on success, false if file doesn't exist
local ok = Window:LoadConfig("default")
if not ok then
    print("No saved config found, using defaults")
end
✓
Call LoadConfig after you have added all your widgets. Loading before widgets are registered means their values have nowhere to go.
20

Auto-Save

When enabled, the config is saved automatically every time any keyed widget's value changes. Call it after adding all widgets and after the initial LoadConfig.

Recommended pattern
-- 1. Add all your widgets with keys
Tab:AddCheckbox("God Mode", false, onGodMode, "godmode")
Tab:AddSliderInt("Walk Speed", 0, 200, 16, onSpeed, "walkspeed")
Tab:AddCombo("Team", {"Red","Blue"}, "Red", onTeam, "team")

-- 2. Load saved config (applies values and fires callbacks)
Window:LoadConfig("default")

-- 3. Enable auto-save so any future change is persisted
Window:EnableAutoSave("default")

Config API

Full reference for all config-related methods on the Window object.

config
Window:SaveConfig(name?: string) → boolean

Saves all keyed widget values to ImGuiLib/<name>.json. Creates the folder if it doesn't exist. Returns true on success, false on filesystem error. Defaults to "default" if no name given.

config
Window:LoadConfig(name?: string) → boolean

Loads ImGuiLib/<name>.json and applies all stored values to matching keyed widgets, firing each widget's callback. Returns false if the file doesn't exist or fails to parse.

config
Window:EnableAutoSave(name?: string)

From this point on, every time any keyed widget changes it calls SaveConfig(name) automatically. Call this after adding all widgets and after the initial LoadConfig.

config
Window:DisableAutoSave()

Turns off auto-save. You can still call SaveConfig manually.

config
Window:DeleteConfig(name?: string) → boolean

Deletes ImGuiLib/<name>.json from the filesystem. Returns true on success.

config
Window:ListConfigs() → {string}

Returns a table of all saved config names (without the .json extension) found in the ImGuiLib/ folder.

Example
local configs = Window:ListConfigs()
-- { "default", "preset_a", "preset_b" }
for _, name in ipairs(configs) do
    print(name)
end

Theming

Pass a table of overrides to Window:SetTheme({}). Keys you don't specify keep their Dear ImGui dark theme defaults.

Example
Window:SetTheme({
    TitleBgActive = Color3.fromRGB(80, 40, 120),  -- purple title bar
    SliderGrab    = Color3.fromRGB(200, 80,  80),  -- red slider fill
    CheckMark     = Color3.fromRGB(80, 200, 120),  -- green checkmarks
})

Keybind

Default toggle key is RightShift. The keybind only fires when GameProcessedEvent is false — it won't trigger while typing in a textbox or using in-game chat.

Example
local Window = ImGui.CreateWindow("Tool", { ToggleKey=Enum.KeyCode.Insert })
Window:SetToggleKey(Enum.KeyCode.F4)  -- or change at runtime

Full Example

A complete script with multiple tabs, keyed widgets, config loading, and auto-save.

Luau
local ImGui = loadstring(game:HttpGet(
    "https://gist.githubusercontent.com/swapemc/5471fafc4ab020982e882d243fbbdf4d/raw/574f5fd2c654a9e0d20747290e3084bb56dea591/gistfile1.txt"
))()

local Window = ImGui.CreateWindow("My Tool", {
    Width     = 430,
    Height    = 500,
    ToggleKey = Enum.KeyCode.RightShift,
})

-- ── MAIN TAB ──────────────────────────────────────────────
local Main = Window:AddTab("Main")

Main:AddText("Hello, world!")
Main:AddSeparator()

Main:AddCheckbox("God Mode", false, function(v)
    local hum = game.Players.LocalPlayer.Character.Humanoid
    hum.MaxHealth = v and math.huge or 100
    hum.Health    = hum.MaxHealth
end, "godmode")

Main:AddSliderInt("Walk Speed", 0, 200, 16, function(v)
    game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = v
end, "walkspeed")

Main:AddSliderFloat("Gravity", 0.0, 5.0, 1.0, function(v)
    workspace.Gravity = v * 196.2
end, "gravity")

Main:AddInputText("Target", "", function(t) print(t) end, "target")

Main:AddCombo("Aim Part", {"Head","Torso","HumanoidRootPart"}, "Head",
    function(v) print(v) end, "aimpart")

Main:AddButton("Rejoin", function()
    game:GetService("TeleportService"):Teleport(game.PlaceId)
end)

-- ── VISUALS TAB ───────────────────────────────────────────
local Visual = Window:AddTab("Visuals")

local esp = Visual:AddCollapsingHeader("ESP", true)
esp:AddCheckbox("Boxes",    true,  function(v) print(v) end, "espboxes")
esp:AddCheckbox("Name Tags", false, function(v) print(v) end, "espnames")
esp:AddColorEdit("ESP Color", Color3.fromRGB(255,0,0),
    function(c) print(c) end, "espcolor")

-- ── SETTINGS TAB ──────────────────────────────────────────
local Settings = Window:AddTab("Settings")

Settings:AddText("Config")
Settings:AddButton("Save Config",  function() Window:SaveConfig("default") end)
Settings:AddButton("Load Config",  function() Window:LoadConfig("default") end)
Settings:AddButton("Delete Config", function() Window:DeleteConfig("default") end)
Settings:AddSeparator()
Settings:AddButton("Destroy GUI", function() Window:Destroy() end)

-- ── INIT: load saved config, then enable auto-save ────────
Window:LoadConfig("default")
Window:EnableAutoSave("default")
